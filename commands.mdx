## Understanding Command Syntax

<Tabs>
  <Tab title="Command Structure">
    Casey commands follow this general structure:
    
    ```
    ,command [required argument] (optional argument)
    ```
    
    For example:
    
    ```
    ,warn @user (reason)
    ```
    
    This would warn the mentioned user with an optional reason.
  </Tab>
  <Tab title="Permissions">
    Each command requires specific permissions to use. These will be listed with each command when the full documentation is available.
    
    <Warning>
    Make sure Casey has the proper permissions in your server to execute commands properly.
    </Warning>
  </Tab>
</Tabs>

## Moderation Commands

<ResponseField name="ban" type="Moderation">
  Bans a user from the server.
  
  **Usage**: `,ban @user (reason)`
</ResponseField>

<ResponseField name="kick" type="Moderation">
  Kicks a user from the server.
  
  **Usage**: `,kick @user (reason)`
</ResponseField>

<ResponseField name="warn" type="Moderation">
  Warns a user and logs the warning.
  
  **Usage**: `,warn @user (reason)`
</ResponseField>

## Configuration Commands

<ResponseField name="prefix" type="Configuration">
  Changes the bot's prefix for your server.
  
  **Usage**: `,prefix set (symbol)`
</ResponseField>
